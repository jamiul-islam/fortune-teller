## File Naming Conventions

- **Components**: PascalCase (e.g., `HeroSection.tsx`, `Button.tsx`)
- **Utilities**: camelCase (e.g., `utils.ts`)
- **Config**: kebab-case (e.g., `fortune-notes.json`)
- **API Routes**: kebab-case folders (e.g., `app/api/webhooks/calendly/route.ts`)

## Component Organization

### Section Components (`components/sections/`)

Full-page sections with 100vh height:

- Each section is a self-contained client component
- Props interface defined at top of file
- Uses shadcn/ui components wrapped with Motion.js
- Imports from `@/components/ui/` for UI primitives

### UI Components (`components/ui/`)

shadcn/ui components added via CLI:

- Generated by `bunx shadcn@latest add <component>`
- Do not manually edit unless customizing
- Use `className` prop for styling overrides
- Wrap with `motion()` for animations

## Import Aliases

## Page Structure

### Landing Page (`app/page.tsx`)

Client component that composes sections:

1. HeroSection (with video background)
2. ConsultationCategoriesSection
3. SocialProofSection
4. TrustSection
5. FinalCTASection
6. FooterSection (in layout)

### Confirmation Page (`app/confirmation/page.tsx`)

Token-based page showing:

- Welcome video
- Booking details
- Navigation to solution page

### Solution Page (`app/solution/page.tsx`)

Token-based page showing:

- Solution card
- Personalized video (if available)
- Random fortune note
- Link to book another consultation

## Styling Approach

- **Tailwind CSS v4** for all styling
- **CSS Variables** for theme colors (defined in `globals.css`)
- **Mystical Premium Palette**: cosmic purples, mystic gold, celestial blue, ethereal pink
- **No custom CSS files** - use Tailwind utility classes
- **Responsive**: Mobile-first approach with breakpoints (sm, md, lg, xl)

## State Management

- **Client State**: React hooks (useState, useRef)
- **Server State**: Supabase queries via MCP tools
- **Form State**: React Hook Form (if needed for complex forms)
- **No global state library** - keep state local to components
